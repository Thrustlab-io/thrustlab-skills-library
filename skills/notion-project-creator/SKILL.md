# Notion Project Creator Skill

You are helping create a complete Notion workspace for a new GTM client by replicating the proven GTM Client Hub Template structure. This skill reads an existing GTM strategy markdown file (generated by gtm-strategy-generator), extracts client information, creates the workspace structure, and populates the GTM Strategy page with the content.

## Prerequisites

**IMPORTANT**: This skill expects that a GTM strategy has already been generated using the gtm-strategy-generator skill. The strategy markdown file should exist at `strategies/[CLIENT_NAME].md`.

## Required Information

The skill takes one argument:
1. **Client name or strategy file path** - Either the client name (will look for `strategies/[CLIENT_NAME].md`) or the full path to the strategy markdown file

The skill will then:
1. Read the strategy markdown file
2. Extract client information from the strategy:
   - Client name (from filename or document)
   - Client website (from "Company Snapshot" or similar section)
   - Product/service description
   - Value proposition
3. Use this information to create the Notion workspace
4. Copy the strategy content to the GTM Strategy page

## Strategy File Parsing

The skill must extract client information from the GTM strategy markdown file to populate the Notion workspace.

### Expected Strategy File Location:
- Default path: `strategies/[CLIENT_NAME].md`
- Or user-provided full path

### Information to Extract:

Look for these sections in the strategy file (typically in the first section):

1. **Company Snapshot / Company Overview:**
   - Client name (e.g., "Client Name: Acme Corp")
   - Website URL (e.g., "Website: https://acme.com")
   - Product/Service description
   - Value proposition
   - Primary use cases

2. **Market Information:**
   - Primary geography
   - Primary industries
   - Target personas/titles

3. **Additional Context:**
   - Average deal size/ACV
   - Sales motion (SaaS/Services/Hybrid)

### Parsing Strategy:

```markdown
1. Read the entire markdown file
2. Look for patterns like:
   - "Client Name:" or "Company:" followed by the name
   - "Website:" or "URL:" followed by the URL
   - First H1 heading may contain company name
3. Extract all relevant information
4. Use extracted info to populate Notion workspace placeholders
5. Copy the entire strategy content to the GTM Strategy Notion page
```

### Fallback Behavior:

If information cannot be extracted:
- Use the filename (CLIENT_NAME from strategies/CLIENT_NAME.md)
- Ask user to confirm or provide missing information
- Proceed with placeholders for any missing data

## Error Handling

### Strategy File Not Found:
- Check if file exists at `strategies/[CLIENT_NAME].md`
- Try variations of the client name (with/without spaces, underscores vs hyphens)
- Ask user to provide the correct file path or client name
- Suggest running `/gtm-strategy-generator [website]` first if strategy doesn't exist
- Do NOT proceed with workspace creation if strategy file cannot be found

### Strategy File Parsing Errors:
- If client information cannot be extracted, ask user to provide it manually
- Use filename as fallback for client name
- Proceed with best-effort information extraction

### Notion API Errors:
- Check authentication status (`/mcp` command)
- Verify permissions to create pages
- Provide clear error messages with troubleshooting steps
- Offer to retry or create pages manually
